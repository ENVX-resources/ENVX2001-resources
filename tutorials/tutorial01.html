1 RStudio Projects

RStudio Projects help you keep your work organised and reproducible. While not strictly necessary, they make your life easier, especially when working on multiple project.

1.1 Why Use Projects?

RStudio Projects help with organisation by keeping all project files together in one directory. They also ensure reproducibility by making your code work consistently across different computers. Think of them as a way to ‚Äútell‚Äù RStudio that all your work for a particular project is in one place.

At its heart, an RStudio Project is simply a text file with the .Rproj extension. This file lives in your folder that you‚Äôre working in, and tells RStudio to treat that folder as a project. Once created, RStudio manages the project for you and you never have to edit the .Rproj file directly.

1.2 Setting up a Project

Let‚Äôs set up a new project for ENVX2001 in RStudio.

  1. Open RStudio
  2. Click File -> New Project‚Ä¶
  3. Select either New Directory or Existing Directory
    • New Directory creates a new folder for your project
    • Existing Directory lets you choose a folder that already exists
  4. Select New Project unless you have a specific project you want to work on (e.g. a Quarto Blog)
  5. Name your project ‚ÄúENVX2001‚Äù or similar, choose where to save it, and click Create Project. You will see other options, but you can ignore them for now.

Give your project a meaningful name, ideally with ‚ÄúENVX2001‚Äù in it so you can identify it later. For example ‚ÄúENVX2001 Labs‚Äù.

1.4 Opening Projects

From File Explorer (PC) or Finder (Mac)

  1. Navigate to your project folder
  2. Double-click the .Rproj file
    • On Windows: Look for the RStudio cube icon
    • On Mac: Look for the .Rproj extension

From Within RStudio

Manually:

  1. Click File -> Open Project...
  2. Navigate to your project folder
  3. Select the .Rproj file
  4. Click ‚ÄúOpen‚Äù

Recent projects:

If you‚Äôve opened the project before, your project may already be open! Check the top-right corner of RStudio ‚Äì if you see your project name, you‚Äôre good to go. Click on the dropdown to switch to recently opened projects.

1.5 Benefits

When in a project, RStudio will remember your working directory, history, and environment. This means you can close RStudio and come back to your project later, and everything will be as you left it. This is especially useful when working on a project over multiple days. Additionally, adding data files to your project folder will make them easier to access in your code as you can use relative paths. For example, instead of using:

read.csv("C:/Users/username/Documents/ENVX2001/data.csv")

you can use:

read.csv("data/data.csv")

The latter is more portable and will work on any computer, as long as the data file is in the data folder, while the former will only work on your computer as it would have your user name on it!

1.6 What‚Äôs next?

Set up an RStudio project separately for your labs and projects. This will make it easier to keep your work organised and reproducible. Otherwise, there isn‚Äôt much else to think about as RStudio will manage the project for you. Just remember to open your project when you start working on it!

2.5 Exercise 4: Try Different Output Formats

Microsoft Word

Try creating a Word document by changing your YAML to:

  docx:
    toc: true
    number-sections: true

You can learn more about Word output options in the Quarto Word documentation.

Typst (Advanced Option)

Typst is a modern alternative to LaTeX/PDF, but it requires some learning to customise effectively. If you‚Äôre interested in exploring it:

format:
  typst:
    toc: true

Learn more about Typst in: - Quarto‚Äôs Typst documentation - Official Typst documentation

Multiple Output Formats

You can create multiple output formats simultaneously by specifying them in your YAML header:

format:
  html:
    toc: true
    number-sections: true
    theme: cosmo
  docx:
    toc: true
    number-sections: true

Each format can have its own specific options. For example, you could have different themes for HTML while maintaining consistent structure across all formats.

3.2 Exercise 2: Working with code chunks

Now let‚Äôs focus on code. We will work with data from the palmerpenguins package to visualise and summarise some data. Try running the following code chunks in your document:

```{r}
#| label: setup
#| message: false
```

Never run package installation code (install.packages()) in your Quarto document! Instead, install required packages in the R console:

This ensures your document remains reproducible and prevents installation errors during rendering.

Let‚Äôs break down the chunk options:

  • label: gives your chunk a unique name
  • message: false suppresses package loading messages
  • warning: false suppresses warnings
  • The #| syntax makes options more readable

3.6 Exercise 6: Useful chunk options

Here are some useful chunk options, try them:

  1. Cache computations (great for slow calculations):
```{r}
#| label: slow-calculation
#| cache: true

# Your time-consuming code here
Sys.sleep(2)  # Simulating slow computation
```
  1. Run code but hide everything:
```{r}
#| label: hidden-setup
#| include: false

# This code runs but shows nothing
theme_set(theme_minimal())
```
  1. Show code but don‚Äôt run it:
```{r}
#| label: example-code
#| eval: false

# This code is displayed but not executed
ggplot(data) +
  geom_point()
```

Try changing the values for options like echo, include, and others from true to false (or vice versa). See what happens.

3.7 Quick tips

  1. Use meaningful chunk labels - they help with debugging
  2. Keep chunks focused on single tasks
  3. Cache slow computations with cache: true
  4. Use include: false for setup code you don‚Äôt want to show
